# Runbooks

Runbooks describe repeatable procedures for operating, deploying, recovering, or supporting the project.

Good runbooks include:

- Purpose and scope.
- Prerequisites.
- Exact commands.
- Expected outputs.
- Rollback steps.
- Troubleshooting notes.
- Owner or escalation path.

Recommended runbooks:

- `local-development.md`
- `release.md`
- `deployment.md`
- `incident-response.md`
- `data-refresh.md` for data workflows.
