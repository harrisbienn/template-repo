# Documentation

Use this folder for durable project knowledge.

Recommended sections:

- `architecture/` for system structure, interfaces, and design notes.
- `decisions/` for architecture decision records.
- `runbooks/` for repeatable operational procedures.
- `standards/` for repository, release, coding, and data handling conventions.

Documentation should make future work easier. Prefer concrete commands, assumptions, diagrams, examples, and links over vague descriptions.
