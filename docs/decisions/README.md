# Architecture Decision Records

Use this folder for architecture decision records, often called ADRs.

Create a new ADR when a decision is hard to reverse or future maintainers will ask why it was made.

Recommended filename format:

```text
0001-short-decision-title.md
0002-another-decision.md
```

Each ADR should include:

- Status.
- Context.
- Decision.
- Consequences.
- Alternatives considered.
