# Notebooks

Use notebooks for exploration, analysis explanation, and reviewable narratives.

Recommended workflow:

1. Explore in `notebooks/exploration/`.
2. Move stable logic into `src/`.
3. Keep notebooks focused on orchestration, visualization, and explanation.
4. Record data sources and parameters.
5. Clear large outputs before committing unless the notebook output is the deliverable.

Notebook checkpoints are ignored by Git.
